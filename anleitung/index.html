TERMUX INSTALLIEREN

Termux nicht aus dem Google Play Store laden — diese Version ist veraltet und nicht mehr gepflegt.

Nur F-Droid verwenden! Play Store Version funktioniert nicht zuverlässig.
F-DROID
Offizielle Termux-Quelle
🔗 F-Droid öffnen
2
SPEICHER-ZUGRIFF ERLAUBEN

Damit Termux auf deine Downloads zugreifen kann, einmalig ausführen:

TERMUX
termux-setup-storage

Im Dialog "Zugriff erlauben" antippen.

3
ZIP HERUNTERLADEN + SETUP STARTEN

ZIP-Datei und Setup-Skript in den Downloads-Ordner legen, dann:

SETUP-SKRIPT
termux_setup.sh — alles automatisch
📥 Herunterladen
TERMUX
bash /storage/emulated/0/Download/termux_setup.sh
Das Skript installiert PHP, entpackt alles und richtet den Server ein — dauert ca. 3 Minuten.
4
SERVER STARTEN
TERMUX
bash ~/start_turnier.sh
ADMINhttp://[DEINE-IP]:8080/admin/
ZUSCHAUERhttp://[DEINE-IP]:8080/mobile/
TVhttp://[DEINE-IP]:8080/tv/
Samsung Hotspot blockiert manchmal interne Ports → Externen WLAN-Router (TP-Link ~15€) empfohlen.
🎉

FERTIG!

Server läuft. Zuschauer verbinden sich per QR-Code aus dem Admin-Bereich.

1
XAMPP INSTALLIEREN

XAMPP ist ein kostenloser Webserver-Stack für Windows (Apache + PHP). Einmalig installieren:

XAMPP DOWNLOAD
Apache Friends — kostenlos, Open Source
🔗 Herunterladen

Installer starten und durchklicken. Standardpfad C:\xampp beibehalten.

Nach der Installation öffnet sich das XAMPP Control Panel → Apache auf "Start" klicken.
2
SETUP-SKRIPT AUSFÜHREN

ZIP-Datei und Setup-Skript in denselben Ordner legen (z.B. Desktop):

WINDOWS INSTALLER
windows_setup.bat — alles automatisch
📥 Herunterladen

Rechtsklick auf windows_setup.bat → Als Administrator ausführen

Das Skript findet XAMPP automatisch, entpackt TurnierOffline und erstellt eine Desktop-Verknüpfung.
3
IP-ADRESSE ERMITTELN

Windows-Taste + R → cmd → Enter

CMD
ipconfig

Die Zeile IPv4-Adresse ist deine Netzwerk-IP, z.B. 192.168.1.105

4
LOSLEGEN
ADMINhttp://localhost/TurnierOffline/admin/
ZUSCHAUERhttp://[DEINE-IP]/TurnierOffline/mobile/
TVhttp://[DEINE-IP]/TurnierOffline/tv/
Desktop-Verknüpfung "TurnierOffline starten.bat" öffnet automatisch den Browser und zeigt deine IP an.
🎉

FERTIG!

Apache läuft, TurnierOffline ist erreichbar. Zuschauer per QR-Code verbinden.

1
ZIP + INSTALLER HERUNTERLADEN

TurnierOffline ZIP und das Linux-Setup-Skript in denselben Ordner legen (z.B. Downloads):

LINUX INSTALLER
linux_setup.sh — Ubuntu, Debian, openSUSE
📥 Herunterladen
Erkennt automatisch: Ubuntu/Debian (apt), openSUSE (zypper), Fedora (dnf).
2
INSTALLER AUSFÜHREN

Terminal öffnen und ausführen:

BASH
sudo bash linux_setup.sh

Das Skript installiert PHP, richtet einen systemd-Service ein (startet automatisch beim Booten) und erstellt einen Desktop-Shortcut.

openSUSE: Falls PHP-Repository fehlt: sudo zypper addrepo https://download.opensuse.org/repositories/devel:languages:php/openSUSE_Leap_15.6/ php-repo
3
MODUS WÄHLEN + STARTEN

Nach der Installation öffnet sich das Startmenü automatisch — oder manuell aufrufen:

BASH
bash ~/TurnierOffline_Installer/start.sh
MODUS 1Admin — Browser öffnet, PC normal
MODUS 2Server — Hintergrund, PC gesperrt
MODUS 3TV — Kiosk-Monitor, Laptop frei/gesperrt
Netzwerk-Optionen: DHCP+DNS (turnier.local) oder Captive Portal — direkt im Startmenü wählbar, kein Router nötig.
🎉

FERTIG!

systemd-Service läuft. Startet beim nächsten Boot automatisch.

1
DOCKER DESKTOP INSTALLIEREN

Docker Desktop läuft auf Windows, macOS und Linux — einmal installiert, überall gleich:

DOCKER DESKTOP
docker.com — kostenlos für private Nutzung
🔗 Herunterladen

Nach Installation Docker Desktop starten und warten bis es läuft (Wal-Symbol in Taskleiste).

2
DOCKER INSTALLER AUSFÜHREN

ZIP + Setup-Skript in denselben Ordner:

DOCKER INSTALLER
docker_setup.sh — baut Image, startet Container
📥 Herunterladen
BASH / GIT BASH
bash docker_setup.sh
Beim ersten Mal dauert das Build ca. 2–3 Minuten. PHP 8.3 Alpine Image (50MB).
3
VERWALTEN
BASH
bash docker_start.sh          # Starten
bash docker_start.sh stop     # Stoppen
bash docker_start.sh update   # Update (Daten bleiben)
ADMINhttp://localhost:8080/admin/
ZUSCHAUERhttp://[DEINE-IP]:8080/mobile/
TVhttp://[DEINE-IP]:8080/tv/
Datenbank und Uploads sind in Docker Volumes — bleiben auch nach Updates erhalten.
🎉

Container läuft. Startet automatisch mit Docker Desktop.
